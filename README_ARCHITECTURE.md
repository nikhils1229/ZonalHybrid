# Architecture Mapping: CAN vs CAN-FD in FiCo4OMNeT

In the unified `FiCo4OMNeT` source code, CAN and CAN-FD share the same C++ classes and NED modules, with behavior branching based on specific flags and parameters.

## 1. Message Differentiation (`CanDataFrame.msg`)
Both protocols use the `CanDataFrame` packet type. The distinction is made via:
- `bool isCanFd`: If `true`, the frame is treated as CAN-FD.
- `bool brs`: Bit Rate Switch flag. If `true`, the data phase is transmitted at the higher `dataBitrate`.
- `int dlc`: For CAN-FD, DLC values 9-15 map to payload sizes 12-64 bytes (handled in `CanFdUtil.h`).

## 2. Logic and Timing (`CanFdUtil.h` & `CanBusLogic.cc`)
- **Bitrate Switching**: `CanFdUtil::calculateTransmissionTime` implements the dual-bitrate logic. It calculates the duration of the arbitration phase using the nominal bandwidth and the data phase using the `dataBitrate`.
- **Bus Configuration**: The `canBus` NED module (and its `CanBusLogic` submodule) defines a `dataBitrate` parameter (defaulting to 5Mbps). Standard CAN simulations typically leave this at default or ignore it, while CAN-FD configurations explicitly set it in `omnetpp.ini`.
- **Automatic Detection**: `CanTrafficSourceAppBase.cc` automatically sets `isCanFd = true` and `brs = true` if the configured payload length exceeds 8 bytes.

## 3. Codebase Map
- `src/fico4omnet/linklayer/can/messages/CanDataFrame.msg`: Defines the unified frame structure.
- `src/fico4omnet/utilities/CanFdUtil.h`: Contains the "source of truth" for CAN-FD protocol rules (DLC-to-length mapping and stuffed bit length calculations).
- `src/fico4omnet/bus/can/CanBusLogic.cc`: The central arbiter that respects the `dataBitrate` for frame duration.
- `src/fico4omnet/linklayer/can/CanPortOutput.cc`: Calculates transmission delay by querying the bus for its `dataBitrate`.

## 4. Zonal Hybrid Model Integration
The Zonal Hybrid model leverages this by defining heterogeneous buses in the topology. A "Standard CAN" bus and a "CAN-FD" bus use the same NED types but differ in their `dataBitrate` and the `isFd` flags of the messages generated by attached nodes. Gateways in the zonal architecture perform protocol translation by toggling these flags and re-calculating CRC/timing via the aforementioned utilities.
